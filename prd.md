# 구구단 산성비 게임 PRD (Product Requirements Document)

## 1. 제품 개요

### 1.1 제품명
**구구단 산성비 (Multiplication Rain)**

### 1.2 목적
초등학교 2학년 학생들이 재미있게 곱셈구구를 학습하고 연습할 수 있는 교육용 게임

### 1.3 타겟 사용자
- 주 사용자: 초등학교 2학년 학생 (만 7-8세)
- 부 사용자: 학부모, 교사

### 1.4 핵심 가치
- 게임을 통한 자연스러운 구구단 학습
- 반복 학습을 통한 암기력 향상
- 즉각적인 피드백으로 학습 동기 부여

## 2. 게임 메커니즘

### 2.1 기본 게임 플레이
1. 화면 상단에서 곱셈 문제가 적힌 물방울이 천천히 떨어진다
2. 화면 하단에 여러 개의 정답 바구니가 배치되어 있다
3. 플레이어는 키보드(숫자 키) 또는 마우스 클릭으로 정답을 선택한다
4. 정답을 맞추면 물방울이 사라지고 점수를 획득한다
5. 오답을 선택하거나 물방울이 바닥에 닿으면 생명이 감소한다

### 2.2 게임 요소

#### 물방울 (문제)
- **표시 내용**: 곱셈 문제 (예: "3 × 4", "7 × 6")
- **낙하 속도**: 레벨에 따라 점차 증가
- **색상**: 난이도별로 구분
  - 초록색: 2, 3, 4단 (쉬움)
  - 파란색: 5, 6, 7단 (보통)
  - 보라색: 8, 9단 (어려움)

#### 정답 바구니
- 화면 하단에 4-6개의 바구니 배치
- 각 바구니에 서로 다른 숫자 표시
- 정답 1개 + 오답 3-5개

#### 생명 시스템
- 시작 시 하트 5개 제공
- 오답 또는 놓친 문제마다 하트 1개 감소
- 하트가 0이 되면 게임 오버

#### 점수 시스템
- 정답 1개당 10점
- 연속 정답 시 콤보 보너스 (×1.5, ×2.0)
- 빠르게 정답을 맞출수록 추가 보너스

## 3. 게임 모드

### 3.1 연습 모드
- **목적**: 특정 단을 집중 연습
- **특징**:
  - 원하는 구구단(2-9단) 선택 가능
  - 생명 무제한
  - 낙하 속도 조절 가능 (느림/보통/빠름)
  - 시간 제한 없음

### 3.2 도전 모드
- **목적**: 전체 구구단 실력 테스트
- **특징**:
  - 2단부터 9단까지 랜덤 출제
  - 생명 5개 (고정)
  - 레벨업 시스템 (10문제마다 레벨 상승)
  - 레벨이 오를수록 낙하 속도 증가
  - 최고 점수 기록

### 3.3 시간 도전 모드
- **목적**: 제한 시간 내 최대한 많은 문제 풀기
- **특징**:
  - 60초 / 120초 선택 가능
  - 생명 무제한
  - 빠른 낙하 속도
  - 타임 어택 방식

### 3.4 친구 대결 모드 (선택 사항)
- 같은 기기에서 2명이 번갈아가며 플레이
- 같은 조건에서 누가 더 높은 점수를 얻는지 경쟁

## 4. 학습 지원 기능

### 4.1 학습 현황 대시보드
- 각 단별 정답률 표시 (차트)
- 자주 틀리는 문제 분석
- 총 플레이 시간 및 문제 풀이 수
- 주간/월간 학습 진도 그래프

### 4.2 구구단 표 보기
- 게임 외부에서 2-9단 구구단 표 제공
- 각 단 클릭 시 상세 보기
- 소리로 들어보기 기능 (TTS)

### 4.3 칭찬 및 보상 시스템
- 목표 달성 시 스티커/배지 획득
  - "2단 마스터", "완벽한 콤보", "빠른 손가락"
- 캐릭터 커스터마이징 아이템 제공
- 레벨업 시 축하 애니메이션

### 4.4 학부모/교사용 리포트
- 학습 시간 및 빈도
- 단별 숙련도 분석
- 성장 추이 그래프
- 추천 학습 가이드

## 5. UI/UX 요구사항

### 5.1 화면 구성

#### 메인 화면
- 큰 글씨의 메뉴 버튼 (터치하기 쉽게)
- 귀여운 캐릭터 애니메이션
- 배경 음악 on/off 버튼
- 학습 현황 요약 표시

#### 게임 화면
- 상단: 점수, 레벨, 콤보 표시
- 중앙: 물방울이 떨어지는 영역 (넓은 공간)
- 하단: 정답 바구니 (큰 버튼)
- 우측 상단: 남은 생명 (하트 아이콘)
- 좌측 상단: 일시정지 버튼

#### 결과 화면
- 획득한 점수 및 정답 개수
- 별점 평가 (★★★)
- 획득한 배지 표시
- "다시 하기" / "메인으로" 버튼

### 5.2 디자인 가이드라인
- **색상**: 밝고 따뜻한 파스텔톤
- **폰트**: 큰 글씨, 가독성 높은 둥근 서체
- **아이콘**: 직관적이고 귀여운 일러스트
- **애니메이션**: 부드럽고 과하지 않은 움직임
- **사운드**: 경쾌하고 긍정적인 효과음

### 5.3 접근성
- 색맹 사용자를 위한 색상 외 구분 요소 (아이콘, 패턴)
- 터치/클릭 영역 충분히 크게 (최소 44×44px)
- 텍스트 크기 조절 옵션
- 소리 끄기 옵션

## 6. 기술 요구사항

### 6.1 플랫폼
- **우선순위 1**: 웹 브라우저 (PC, 태블릿)
- **우선순위 2**: 모바일 앱 (iOS, Android)
- **우선순위 3**: 오프라인 실행 가능

### 6.2 성능
- 초기 로딩 시간: 3초 이하
- 입력 반응 시간: 100ms 이하
- 프레임율: 60fps 유지
- 메모리 사용량: 최소화

### 6.3 데이터 저장
- 로컬 저장소 활용 (LocalStorage/IndexedDB)
- 클라우드 백업 옵션 (선택 사항)
- 개인정보 최소 수집 (이름/닉네임만)

### 6.4 기술 스택 제안
- **프론트엔드**: HTML5, CSS3, JavaScript (React/Vue)
- **게임 엔진**: Phaser.js 또는 PixiJS
- **애니메이션**: GreenSock (GSAP)
- **사운드**: Howler.js

## 7. 콘텐츠 요구사항

### 7.1 문제 은행
- 각 단(2-9단) × 9문제 = 총 72개 기본 문제
- 순서 랜덤화
- 중복 최소화 알고리즘

### 7.2 오답 생성 규칙
- 정답 ± 1-10 범위 내 숫자
- 다른 단의 정답 혼합
- 완전히 무작위 숫자 (10% 비율)

### 7.3 사운드 에셋
- 배경 음악 (밝고 경쾌한 BGM)
- 정답 효과음 (딩동!)
- 오답 효과음 (부드러운 실패음)
- 레벨업 효과음
- 게임 오버 음악

### 7.4 비주얼 에셋
- 캐릭터 스프라이트 (최소 3가지 감정)
- 물방울 애니메이션
- 배경 이미지 (3가지 테마)
- 아이콘 세트
- 배지/스티커 (최소 15개)

## 8. 성공 지표 (KPI)

### 8.1 학습 효과
- 각 단별 정답률 70% 이상 달성률
- 1주일 연속 사용률
- 평균 세션 시간: 10-15분

### 8.2 사용자 참여
- 일일 활성 사용자 (DAU)
- 주간 활성 사용자 (WAU)
- 평균 세션 수/주
- 게임 완료율 (게임 오버까지 도달)

### 8.3 만족도
- 사용자 평가 4.0/5.0 이상
- 학부모/교사 추천률 80% 이상
- 재방문율 60% 이상

## 9. 개발 일정 (예상)

### Phase 1: MVP (4주)
- Week 1-2: 기본 게임 메커니즘 구현
  - 물방울 낙하 시스템
  - 정답 입력 및 판정
  - 점수/생명 시스템
- Week 3: 연습 모드 구현
- Week 4: 기본 UI/UX 및 테스트

### Phase 2: 핵심 기능 (4주)
- Week 5: 도전 모드 구현
- Week 6: 학습 현황 대시보드
- Week 7: 보상 시스템 및 애니메이션
- Week 8: 사운드 통합 및 최적화

### Phase 3: 고도화 (2주)
- Week 9: 시간 도전 모드, 추가 기능
- Week 10: 전체 테스트 및 버그 수정

### Phase 4: 출시 준비 (2주)
- Week 11: 사용자 테스트 (초등학생 대상)
- Week 12: 피드백 반영 및 최종 릴리스
